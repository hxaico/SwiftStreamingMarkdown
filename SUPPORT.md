# Support

This repository is in open-source release preparation and does not have versioned public support channels yet.

- Use GitHub issues for public bugs, feature requests, and documentation gaps after the repository is published.
- Use the repository security advisory flow for suspected vulnerabilities. Do not open public issues for unpatched security problems.
- Include platform, package version or commit, a minimal markdown fixture, and validation steps when reporting renderer issues.

Before the first public release, support ownership, response expectations, and package support windows must be confirmed with the maintainers and OSPO.
